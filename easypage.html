簡単な使い方

GitHub アカウントの作り方

  • githubにアクセス
  • username, mailaddress, password, を入力して sign up for Githubをクリック
  • 確認メール内のリンクをクリック

ssh鍵の作り方

  • GitBashひらく

  • 鍵のペアの生成

    $ssh-keygen -t rsa -C "自分のメールアドレス"

  • 公開鍵の内容をコピーする
  • GitHubのAccount Settingsのページを開く
  • 左側のメニューからSSH Keysのページを開きます
  • Add SSH Key のボタンをクリック
  • Titleはおまかせ
  • Keyにコピーしたものを貼り付け
  • Add Key ボタンをクリック

リポジトリの作り方

  • ユーザーのトップページのNew　Repositoryのボタンをクリック
  • Repository nameを入力
  • Create　repositoryをクリック

ローカルへの落とし方

  • Git Bashでgit clone [リポジトリのurl]を実行

コミットのやり方

  • git add [file名]　でインデックスに追加する
  • git commit -m "メッセージ" でコミット

pushの仕方

  • git push -u origin masterを実行

ブランチモデル

  • git flow

    masterブランチ

    リリース用のブランチ

    developブランチ

    開発用ブランチ。リリースできると判断されたらmasterへマージされる

    featureブランチ

    機能追加用ブランチ。developから分岐して、developへマージする

    hotfixブランチ

    バグ修正などに使われる。masterから分岐して、masterとdevelopへマージする

    releaseブランチ

    リリース準備用のブランチ

  • girhub flow

    git flow を簡単にしたもの

    developブランチが無くなり、プルリクエストが用いられる。

    masterから分岐して開発する

    完了したらローカルでコミットし、リモートにもプッシュする

    アドバイスやレビューをしてほしい時にはプルリクエストを出す

    必要に応じて修正、コミット、プッシュする

    okが出たらmasterにマージする
